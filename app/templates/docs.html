Instructivo de aprovisionamiento

Tras iniciar sesión como administrador, defina host SSH, rutas y Drive en Integraciones. Los valores guardados sustituyen al .env cuando no están vacíos; el secreto de sesión de cookies se genera en la BD al primer arranque (véase /install en instalación inicial).

1. Firewall en el VPS WHM

Permita SSH solo desde la IP fija del VPS donde corre Dockge (puerto 22 o el que use).

# Ejemplo CSF: csf -a IP_DEL_VPS_BACKUP

2. Clave SSH dedicada

En el servidor de backup genere una clave y copie la pública a /root/.ssh/authorized_keys del WHM.

ssh-keygen -t ed25519 -f ./secrets/whm_rsa -N ""
# Montar ./secrets en el stack (solo lectura)

3. Token WHM (API)

Cree un token con permisos mínimos para operaciones que vaya a automatizar; para restauración automatizada use un token separado con permisos elevados y rote periódicamente.

4. Google Drive + rclone

  1. Instale rclone o use la imagen Docker provista.
  2. rclone config → remote nombre igual a RCLONE_REMOTE (p. ej. gdrive).
  3. Monte el directorio de configuración en el volumen rclone_config del compose.
  4. La jerarquía en Drive seguirá DRIVE_REMOTE_PREFIX (p. ej. WHM03_BACKUP/…).

5. Hook PkgAcct (opcional, aviso por cuenta)

Después de cada cuenta, notifique al worker. Firma HMAC SHA-256 del cuerpo en cabecera X-Signature: sha256=<hex>. Configure WEBHOOK_HMAC_SECRET igual en WHM y en la app.

#!/bin/bash
# /root/hooks/pkgacct_notify.sh — llamar desde hook cPanel postFinalize
BODY='{"user":"'"$user"'","tarball":"'"$tarball"'"}'
SIG=$(echo -n "$BODY" | openssl dgst -sha256 -hmac "$WEBHOOK_SECRET" | awk '{print $2}')
curl -sS -X POST "https://SU_VPS_BACKUP/webhooks/pkgacct" \
  -H "Content-Type: application/json" \
  -H "X-Signature: sha256=$SIG" \
  -d "$BODY"

Ajuste variables user/tarball según el entorno real del hook de cPanel (consulte la documentación oficial de hooks PkgAcct).

6. Webhook público

Exponga https://…/webhooks/pkgacct detrás de TLS (Caddy/Traefik). Sin TLS en producción no es recomendable.
